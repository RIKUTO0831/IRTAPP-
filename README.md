## 💡 プロダクトの概要
「営業T検査」は、最新のIRT（項目応答理論）を用いた精緻な適性把握と、生成AI（Gemini）による動的な営業ロールプレイを組み合わせた新しい適性検査アプリケーションです。

従来の性格検査のように「対策して点数を良く見せる」ことを防ぎ、ユーザー自身が対話を通じて**「自分の本当の強み・課題・営業職との相性」に気づき、自己理解を深める**ことを目的としています。

## 🎯 開発の背景と課題（Why）
**「新卒の3年内離職率」**という社会課題に対し、企業と就活生のミスマッチを防ぐために開発しました。

* **既存の適性検査の限界:** 多くの企業が同じ検査を利用しているため、学生側が「正解」を対策できてしまい、本来の適性が見抜けない状態にあります。
* **ミスマッチの発生:** 実際の営業現場で求められる力を理解しないまま入社し、「思っていた仕事と違う」という早期離職に繋がっています。

本プロダクトは、学生を一方的に「評価・選考する」のではなく、対話と体験を通じて「職種選択や成長のきっかけを作る」UXを目指しています。

## ✨ 主な機能と体験フロー（What）
本アプリは大きく4つのステップで構成されています。

1. **基本診断（IRTによるパラメータ推定）**
   * 20問の固定質問に対し「はい／いいえ」で回答。
   * 単純な合計点ではなく、質問ごとの「答えやすさ（困難度パラメータ）」を考慮し、5つの軸でユーザーの潜在的な特性を算出します。
2. **深掘り診断（AIによる動的質問生成）**
   * 基本診断の結果をもとに、AIがユーザーの傾向に合わせた追加質問（5問）を即座に生成し、より深い傾向を測定します。
3. **AIロールプレイ（実践シミュレーション）**
   * IT・人材・保険・メーカーなど、志望する業界と商談相手を選択。
   * ユーザーの特性パラメータに合わせて、Geminiが動的に「難易度を調整した営業シナリオ」を生成し、5ターンのロールプレイを行います。
4. **総合フィードバック**
   * 診断結果のレーダーチャート化と、ロールプレイの選択履歴を踏まえたAIからの詳細なアドバイスを提示します。

## 🛠 技術的な挑戦とこだわり（How）
### 1. 項目応答理論（IRT）のシステム実装
ゼミで専攻しているIRTを実際のアプリケーションロジックに落とし込みました。
単純に「はい」の数を数える古典的テスト理論ではなく、ロジスティック曲線に基づく確率モデルを用いています。

$P(u_i = 1 | \theta) = \sigma(a_i(\theta - b_i))$
* ※ $\theta$: ユーザーの潜在特性, $a_i$: 識別力, $b_i$: 困難度

ハッカソン用の仮想データから推定したパラメータを用い、最尤推定によってユーザーの能力値を算出するアルゴリズムをゼロから実装しました。

### 2. Generative AI (Gemini 2.5 Flash) の統合
推論速度の速いGemini 2.5 Flash APIを採用し、ユーザーの待ち時間を最小限に抑えています。単なるチャットボットではなく、「IRTの推定結果（ユーザーの特性レベル）」をプロンプトのコンテキストに含めることで、**ユーザーごとに絶妙に判断に迷う難易度のロールプレイ**を生成する仕組みを構築しました。

### 3. 堅牢なエラーハンドリング
通信エラーやAPIの制限が発生した場合でも、アプリケーションがクラッシュすることなく最後まで体験できるよう、フォールバック用の固定質問やエラー時専用の商談シナリオを実装しています。
